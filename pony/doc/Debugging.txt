Отладка программ
================

Иногда написанный код ведет себя совсем не так, как мог предполагать разработчик.
Например, веб-приложение генерирует страницу, которая содержит не ту информацию, которую ожидал
увидеть разработчик.
Мы можем видеть только результат работы программы в виде сгенерированной страницы.
Чтобы понять, почему результат именно такой, нужно убедиться, что процесс построения страницы
соответствует ожидаемому сценарию. Этот процесс называется отладкой.

Пони позволят наблюдать что происходит внутри функции-обработчика, когда та генерирует страницу.

Чтобы дать понять Пони, что мы хотим перейти в режим отладки и наблюдать процесс построения страницы,
нужно передать параметр debug=true в желаемый URL.

Рассмотрим пример.
Наберем следующий код в интерактивном режиме(__подумать как сделать, чтобы читающий мог сразу запустить этот код
                                             может добавить сюда линк на файл .py)

    from pony.main import *
    
    @http
    def page():
        a = 1
        b = 2
        c = a + b
        return html("result: c=$c")

    http.start()

Теперь откроем http://localhost:8080/page в браузере. Мы увидим сгенерированную страницу:

    __картинка__

Давайте теперь посмотрим на генерацию той же страницы в режиме отладки.
Для этого нам нужно открыть тот же самый URL, только добавить GET параметр debug=true.
URL будет выглядеть так: http://localhost:8080/page?debug=true

    __картинка__
